# 光构生产上线检查清单

[English](../../en/deployment/checklist.md) · [部署总览](./index.md) · [文档中心](../../README.md)

本清单用于把本地可用版本部署为公开产品。每一项都应由实际运营者确认，不要直接把本地测试配置用于公网。

部署步骤请按选定路线执行：[Docker Compose 部署](./docker.md)或[传统服务器手动部署](./manual.md)。本清单用于最终验收，不能替代部署教程。

## 1. 账号与访问

- [ ] 使用全新数据库启动后，立即登录管理后台并修改 `root` 的初始密码 `123456`。
- [ ] 创建独立的日常管理员账号，不与普通用户共用账号。
- [ ] 关闭不使用的注册方式、OAuth 服务和管理入口。
- [ ] 验证普通用户无法进入管理后台或修改其他用户额度。
- [ ] 对登录、注册、兑换码和生成接口设置合理限流。

## 2. 密钥与环境变量

- [ ] 为 `SESSION_SECRET` 和 `USER_TOKEN_SECRET` 生成不同的、至少 32 位的高熵随机值。
- [ ] 将真实 `.env` 放在服务器的受限目录，不提交到 Git，不写入前端构建变量。
- [ ] 确认所有上游 API Key 只保存在后端渠道配置中。
- [ ] 若测试期间使用过真实 API Key，检查调用记录并在必要时轮换。
- [ ] 固定保存 `USER_TOKEN_SECRET`；修改它会导致已有用户令牌失效。

PowerShell 可用以下方式分别生成随机值：

```powershell
[Convert]::ToHexString([Security.Cryptography.RandomNumberGenerator]::GetBytes(32)).ToLower()
```

## 3. 域名与网络

- [ ] 使用 HTTPS，并将 HTTP 永久重定向到 HTTPS。
- [ ] 前端与 `/api`、`/v1` 使用同一域名，减少跨域 Cookie 风险。
- [ ] 公网只开放 80/443；数据库、Redis 和后端管理端口不直接暴露。
- [ ] 使用防火墙或安全组限制服务器管理端口。
- [ ] 配置可信代理、真实客户端 IP 和请求体大小限制。

## 4. 数据库、缓存与备份

- [ ] 生产环境使用 MySQL 或 PostgreSQL，不使用单机 SQLite 承载公开业务。
- [ ] 数据库账户使用独立强密码，并只授予所需数据库权限。
- [ ] 多实例部署时使用共享数据库和 Redis，并确保所有实例使用同一组签名密钥。
- [ ] 为数据库建立自动备份、异地副本和恢复演练。
- [ ] 将 `GOUO_ASSET_DIR` 放在持久卷中，并把光构资产目录与数据库放入同一套备份和恢复演练；只恢复其中一方会造成作品记录与文件不一致。
- [ ] 单实例部署确认资产目录可写；多实例部署前改用共享文件系统，不能让各实例使用独立本地目录。
- [ ] 明确日志、账户、订单和调用记录的保留周期。

## 5. 渠道与计费

- [ ] 在管理后台配置可用的 OpenAI 兼容渠道和 `gpt-image-2` 模型映射。
- [ ] 分别实测图片生成、编辑和变体接口。
- [ ] 根据上游真实账单核对 `GOUO_IMAGE_PRICE_CNY`，确保固定售价覆盖成本。
- [ ] 验证成功请求扣费、失败请求退款和余额不足拒绝流程。
- [ ] 设置单用户并发、每分钟请求数和每日消费上限，防止滥用和意外账单。

## 6. 充值与支付

- [ ] 未配置正式支付渠道前，仅开放人工充值或兑换码。
- [ ] 支付回调必须使用支付平台签名验证，回调地址必须为 HTTPS。
- [ ] 分别测试成功、失败、取消、重复通知和延迟通知。
- [ ] 对账确认订单金额、用户额度和支付平台流水一致。
- [ ] 发布清晰的价格、退款和发票规则。

## 7. 内容、安全与隐私

- [ ] 配置并验证上游内容安全策略，保留 request ID 供申诉和排查。
- [ ] 确认日志不记录完整密码、Cookie、授权头、API Key 或图片私密链接。
- [ ] 发布用户协议、隐私政策、内容规范和侵权投诉渠道。
- [ ] 说明提示词和参考图会被发送到上游服务，以及各类数据的保存位置和期限。
- [ ] 提供账号停用、数据导出和删除的处理流程。

## 8. 发布验证

- [ ] 执行 `npm run build` 和 `npm test`。
- [ ] 执行后端核心测试，并人工回归注册、登录、退出、充值、生成、退款和用户中心。
- [ ] 使用全新浏览器验证首次访问流程，使用普通账号验证权限边界。
- [ ] 用已有本地画廊的测试账号验证自动迁移、刷新续传、跨浏览器恢复、回收站和空间不足提示。
- [ ] 配置健康检查、错误率、上游延迟、余额异常和支付异常告警。
- [ ] 准备回滚步骤，保留上一版本镜像和数据库迁移前备份。

完成以上项目后，再将域名解析到公开入口并逐步放量。首次发布建议先使用少量测试账号观察一段时间，再开放注册和充值。
